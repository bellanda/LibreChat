**Escopo**: Sistema prompt + Tool schemas + Tool choice logic

## 🔄 VISÃO GERAL (Fluxo da IA)

```
┌─────────────────────────────────────────────────────────────┐
│ 1. AGENT INITIALIZATION                                     │
│    - Agent carrega de BD/config                             │
│    - Define tools = ['execute_code', 'web_search', ...]    │
└────────────────────┬────────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────────┐
│ 2. TOOL LOADING (api/app/clients/tools/util/handleTools.js)│
│    - Para cada tool, carrega:                              │
│      a) Tool constructor (com schema)                      │
│      b) Descrição legível (toolContext/description)        │
│      c) Auth valores (API keys, etc)                       │
│    - Retorna: loadedTools[] + toolContextMap{}             │
└────────────────────┬────────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────────┐
│ 3. SYSTEM PROMPT ASSEMBLY (packages/api/src/agents/run.ts) │
│    - systemMessage = toolContextMap values (descrições)   │
│    - Concatena com agent.instructions                      │
│    - Resultado = systemContent que vai pro LLM             │
└────────────────────┬────────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────────┐
│ 4. RUN CREATION (@librechat/agents - Run.create)           │
│    - tools: loadedTools[] (com schemas LangChain)         │
│    - instructions: systemContent (prompt com descrições)   │
│    - Cria LangGraph com ferramentas registradas            │
└────────────────────┬────────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────────┐
│ 5. MODEL INVOCATION (LangChain/LLM)                        │
│    - LLM vê: system prompt + tool schemas                  │
│    - LLM decide: use esta ferramenta? Sim → qual?         │
│    - Retorna: tool_call com nome + argumentos              │
└────────────────────┬────────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────────┐
│ 6. TOOL EXECUTION (@librechat/agents - graph.invoke)       │
│    - Chama executeTool(tool_call)                          │
│    - Tool executa (sandbox POST /exec)                     │
│    - Retorna resultado                                      │
└────────────────────┬────────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────────┐
│ 7. RESPONSE LOOP (até LLM dizer "fim")                     │
│    - LLM vê resultado da ferramenta                        │
│    - Continua raciocínio ou chama outra ferramenta         │
│    - Quando pronto, responde ao usuário                    │
└─────────────────────────────────────────────────────────────┘
```

## 📍 PASSO 2: TOOL LOADING

**Arquivo**: `api/app/clients/tools/util/handleTools.js` (linhas 265-291)

### Para `execute_code`:

```javascript
for (const tool of tools) {
  if (tool === Tools.execute_code) {
    requestedTools[tool] = async () => {
      // 1. Carrega auth values
      const authValues = await loadAuthValues({
        userId: user,
        authFields: [`${EnvVar.CODE_API_KEY}||SANDBOX_API_KEY`],
      });
      
      // 2. Carrega API key
      const codeApiKey = authValues[EnvVar.CODE_API_KEY] ?? authValues.SANDBOX_API_KEY;
      
      // 3. "Primes" arquivos (prepara para sandbox)
      const { files, toolContext } = await primeCodeFiles(
        { ...options, agentId: agent?.id },
        codeApiKey,
      );
      
      // 4. Salva contexto (descrição) que vai para o LLM
      if (toolContext) {
        toolContextMap[tool] = toolContext;  // ← Descrição da ferramenta
      }
      
      // 5. Cria tool estruturada (com schema)
      const CodeExecutionTool = createCodeExecutionTool({
        user_id: user,
        files,
        ...authValues,
      });
      
      CodeExecutionTool.apiKey = codeApiKey;
      return CodeExecutionTool;  // ← Tool com schema
    };
  }
  // ... mais ferramentas ...
}
```

## 📍 PASSO 3: SYSTEM PROMPT ASSEMBLY

**Arquivo**: `packages/api/src/agents/run.ts` (linhas 103-113)

```typescript
// Monta o system prompt que vai para a IA
const systemMessage = Object.values(agent.toolContextMap ?? {})
  .join('\n')
  .trim();

// Exemplo do resultado:
const systemMessage = `
# \`execute_code\`: Execute Python or JavaScript code in an isolated sandbox...
# \`web_search\`: 
Current Date: 2026-04-02
1. **Execute immediately without preface** when using \`web_search\`.
2. **Perform ONLY ONE search per user question**...
# \`file_search\`: Performs semantic search across attached documents...
`;

const systemContent = [
  systemMessage,  // ← Descrições das ferramentas
  agent.instructions ?? '',  // ← "You are a helpful AI that..."
  agent.additional_instructions ?? '',  // ← Instruções extras
]
  .join('\n')
  .trim();

// Resultado final:
const systemContent = `
# \`execute_code\`: Execute Python or JavaScript code in an isolated sandbox...

You are a helpful AI assistant that can execute code, search the web, and analyze documents.
Always prefer to use tools when the user asks you to perform tasks.
```

## 📍 PASSO 4: RUN CREATION (LLM Invocation)

**Arquivo**: `packages/api/src/agents/run.ts` (linhas 140-176)

```typescript
const agentInput: AgentInputs = {
  provider: 'openai',  // ou anthropic, google, etc
  reasoningKey: 'reasoning_content',
  agentId: agent.id,
  name: agent.name,
  tools: agent.tools,  // ← Array de tool objects com schemas
  clientOptions: llmConfig,  // ← Model, temperature, etc
  instructions: systemContent,  // ← System prompt montado acima
  maxContextTokens: agent.maxContextTokens,
  useLegacyContent: agent.useLegacyContent ?? false,
};

// Passa para Run.create() da lib @librechat/agents
const run = await Run.create({
  runId,
  graphConfig: {
    signal,
    agents: [agentInput],  // Pode ter múltiplos agentes
    edges: agents[0].edges,  // Handoff entre agentes
  },
  tokenCounter,
  customHandlers,
});
```

**Internamente** (@librechat/agents library):
- Cria um LangGraph com nós para cada ferramenta
- Registra os schemas das ferramentas
- Configura o LLM com tools + instructions

## 📍 PASSO 5: LLM VÊ TUDO E DECIDE

### O que o LLM recebe:

```
System Prompt:
==============
# `execute_code`: Execute Python or JavaScript code in an isolated sandbox...
[Instruções detalhadas sobre quando usar]

# `web_search`: 
Current Date: 2026-04-02
1. **Execute immediately without preface** when using `web_search`.
[... mais instruções ...]

# `file_search`: Performs semantic search across attached documents...

You are a helpful AI that can code, search, and analyze documents.

Tools (schemas OpenAI/Claude format):
====================================
{
  "name": "execute_code",
  "description": "Execute Python or JavaScript code in an isolated environment",
  "input_schema": {
    "type": "object",
    "properties": {
      "code": { "type": "string", "description": "The code to execute" },
      "language": { "enum": ["python", "javascript"] }
    },
    "required": ["code", "language"]
  }
},
{
  "name": "web_search",
  "description": "Search the web for information",
  "input_schema": { ... }
},
...

User Message:
=============
"Analyze this dataset and create a plot"
```

### Como o LLM decide:

```
LLM vê o contexto e pensa:
  "O usuário quer analisar um dataset e criar um plot.
  Eu tenho a ferramenta `execute_code` que pode fazer isso.
  A descrição diz 'Execute Python or JavaScript code in an isolated sandbox'.
  Perfeito! Vou usar `execute_code` com Python."

LLM retorna:
{
  "type": "tool_call",
  "name": "execute_code",
  "id": "call_xyz123",
  "arguments": {
    "code": "import pandas as pd\nimport matplotlib.pyplot as plt\n\ndata = ...",
    "language": "python"
  }
}
```

## 🎯 EXEMPLO REAL: COMO SANDBOX É DESCOBERTO

### Cenário: Usuário pede "Execute este código Python"

```
1️⃣ SETUP:
   Agent é carregado com: agent.tools = ['execute_code', 'file_search']

2️⃣ TOOL LOADING:
   handleTools.js vê 'execute_code'
   → Chama createCodeExecutionTool()
   → Retorna: Tool com schema + description
   → Salva em toolContextMap['execute_code'] = "Execute Python/JS..."

3️⃣ SYSTEM PROMPT:
   run.ts monta:
   systemContent = 
     "# `execute_code`: Execute Python or JavaScript code in an isolated sandbox.
      Use this tool when the user asks you to run code, analyze data, or create files.
      ...
      
      You are a helpful AI that can write and execute code."

4️⃣ LLM RECEBE:
   System: "You can execute code using the `execute_code` tool..."
   Schema: {
     "name": "execute_code",
     "description": "...",
     "input_schema": { "properties": { "code": {...}, "language": {...} } }
   }
   User: "Execute this Python code: print('hello')"

5️⃣ LLM DECIDE:
   "Perfeito! O usuário quer executar código.
    Tenho a ferramenta `execute_code`.
    Vou chamar com o código do usuário e language='python'."

6️⃣ SANDBOX É CHAMADO:
   POST http://sandbox:3082/exec
   Body: { "code": "print('hello')", "language": "python" }
   Resposta: { "stdout": "hello\n", "stderr": "", "exitCode": 0 }

7️⃣ LLM RESPONDE:
   "O código foi executado com sucesso! Output: hello"
```

## 🔧 CUSTOMIZANDO O PROMPT DO SANDBOX

Se você quer mudar como a IA usa o sandbox, edite:

### Opção 1: Descrição Global (System Prompt)

**Arquivo**: `api/server/services/Files/Code/process.js`  
**Função**: `primeCodeFiles()`

```javascript
// Mude o que é retornado em toolContext
toolContext = `
Execute Python or JavaScript code safely in a sandboxed environment.
The code runs in an isolated Docker container with 30s timeout.
Use this for: data analysis, file processing, plotting, calculations.

Available languages:
- Python 3.x (via uv package manager, NOT pip)
- JavaScript/TypeScript (via bun)

Example:
  code = "import pandas as pd; df = pd.read_csv('file.csv'); print(df.head())"
  language = "python"
`;
```

### Opção 2: Instruções do Agent

**Via DB**: MongoDB > agents collection

```javascript
db.agents.updateOne(
  { _id: "agent_id" },
  {
    $set: {
      "instructions": "You are a data scientist...",
      "additional_instructions": "When analyzing code:\n1. Always use pandas\n2. Create visualizations...",
    }
  }
)
```

### Opção 3: Alterar Behavior via Config

**Arquivo**: `librechat.yaml`

```yaml
agents:
  data_scientist:
    model: gpt-4
    tools: [execute_code, file_search, web_search]
    promptPrefix: "You are an expert data scientist..."
    model_parameters:
      temperature: 0.7
      max_tokens: 2000
```

## ⚡ RESUMO: DO DESCOBRIMENTO À EXECUÇÃO

```
Agent.tools = ['execute_code']
       ↓
loadTools() → createCodeExecutionTool()
       ↓
toolContextMap['execute_code'] = "Execute Python or JavaScript..."
       ↓
systemContent = toolContextMap.join('\n') + agent.instructions
       ↓
Run.create({ tools: [...], instructions: systemContent })
       ↓
LLM vê schema + descrição → decide usar execute_code
       ↓
LLM chama: tool_call(name='execute_code', code='...', language='python')
       ↓
LangGraph executa → POST sandbox:3082/exec
       ↓
Sandbox retorna stdout/stderr/files
       ↓
LLM recebe resultado → continua raciocínio ou responde
```
