## `execute_code` (Sandbox)

### Fonte Primária
**Arquivo**: `api/server/services/Files/Code/process.js` → `primeCodeFiles()`

### Prompt Retornado

```
toolContext = "Arquivos de código disponíveis no sandbox" ou similar

(Nota: O prompt exato é dinâmico, vindo de createCodeExecutionTool na lib @librechat/agents)
```

### Descrição da Ferramenta (Schema)
**Vem de**: `createCodeExecutionTool()` do pacote `@librechat/agents`

**O que a IA vê** (aproximado, pois é gerado internamente):
```
Tool Name: execute_code
Description: Execute Python or JavaScript code in an isolated sandbox environment
Input Schema:
{
  "type": "object",
  "properties": {
    "code": {
      "type": "string",
      "description": "The Python or JavaScript code to execute"
    },
    "language": {
      "type": "string",
      "enum": ["python", "javascript"],
      "description": "Programming language: 'python' or 'javascript'"
    }
  },
  "required": ["code", "language"]
}
```

**Como a IA escolhe usá-la**:
- Usuário: "Run this Python code: import pandas; print('hello')"
- IA: "O usuário quer executar código → vou usar execute_code com language='python'"
- Resultado: Tool call com code + language

## `web_search`

### Fonte Primária
**Arquivo**: `api/app/clients/tools/util/handleTools.js` (linhas 326-359)

### Prompt Retornado (em modo normal)

```
# `web_search`:
Current Date: 2026-04-02
1. **Execute immediately without preface** when using `web_search`.
2. **Perform ONLY ONE search per user question** - do not search multiple times before responding to the user.
3. **After the search, begin with a brief summary** that directly addresses the query without headers or explaining your process.
4. **Structure your response clearly** using Markdown formatting (Level 2 headers for sections, lists for multiple points, tables for comparisons).
5. **Cite sources properly** according to the citation anchor format, utilizing group anchors when appropriate.
6. **Tailor your approach to the query type** (academic, news, coding, etc.) while maintaining an expert, journalistic, unbiased tone.
7. **Provide comprehensive information** with specific details, examples, and as much relevant context as possible from search results.
8. **Avoid moralizing language.**
9. **Important**: If you need to search again, wait for the user to ask a new question first. Do not perform multiple searches in the same response.
```

### Prompt Retornado (em auto_mode)

```
# `web_search`: Current Date: 2026-04-02. Use once per question; summarize results clearly.
```

### Descrição da Ferramenta (Schema)
**Vem de**: `createSearchTool()` do pacote `@librechat/agents`

**O que a IA vê** (aproximado):
```
Tool Name: web_search
Description: Search the web for current information
Input Schema:
{
  "type": "object",
  "properties": {
    "query": {
      "type": "string",
      "description": "The search query"
    }
  },
  "required": ["query"]
}
```

**Como a IA escolhe usá-la**:
- Usuário: "What's the latest news about AI?"
- IA: "O usuário quer informação atual → vou usar web_search com query='latest AI news'"
- Resultado: Retorna artigos com sources

## `file_search`

### Fonte Primária
**Arquivo**: `api/app/clients/tools/util/fileSearch.js` (linhas 45-65)

### Prompt Retornado (com arquivos carregados)

```
- Note: Use the `file_search` tool to find relevant information within:
	- documento.pdf
	- relatorio.xlsx (just attached by user)
	- analise.docx
```

### Prompt Retornado (sem arquivos)

```
- Note: Semantic search is available through the `file_search` tool but no files are currently loaded. Request the user to upload documents to search through.
```

### Descrição da Ferramenta (Schema)
**Vem de**: `createFileSearchTool()` em `fileSearch.js`

```
Tool Name: file_search
Description: Performs semantic search across attached documents. 
  This tool analyzes the content of uploaded files to find relevant information, 
  quotes, and passages that best match your query.
Input Schema:
{
  "type": "object",
  "properties": {
    "query": {
      "type": "string",
      "description": "Natural language query to search for in documents"
    }
  },
  "required": ["query"]
}
```

**Como a IA escolhe usá-la**:
- Usuário: "Find the budget from 2024 in my files"
- IA: "O usuário quer buscar em arquivos → vou usar file_search com query='budget 2024'"
- Resultado: Retorna trechos relevantes dos documentos

## MCP Tools (Dinâmico)

### Fonte Primária
**Arquivo**: `api/server/services/MCP/`

### Prompt Retornado

Cada MCP server define seu próprio prompt. Exemplos:
- Se conectar a um MCP server de "fetch URLs": `Tool: fetch - Baixa conteúdo de URLs...`
- Se conectar a um MCP server de "database": `Tool: query_db - Executa queries SQL...`

### Como a IA vê
```
Tool Name: [vem do MCP server]
Description: [vem do MCP server]
Input Schema: [vem do MCP server]
```

## Onde Está Montado o System Prompt Completo?

### Arquivo Principal
**`packages/api/src/agents/run.ts`** (linhas 103-113)

```typescript
const systemMessage = Object.values(agent.toolContextMap ?? {})
  .join('\n')
  .trim();

// Resultado final após join:
`
# \`execute_code\`: [descrição acima]
# \`web_search\`: [descrição acima]
# \`file_search\`: [descrição acima ou vazia se sem arquivos]
`
```

### Depois Concatena Com Agent Instructions

```typescript
const systemContent = [
  systemMessage,                      // ← Descrições das ferramentas
  agent.instructions ?? '',           // ← De agent.promptPrefix ou custom
  agent.additional_instructions ?? '',  // ← Instruções extras
]
  .join('\n')
  .trim();
```

### Exemplo Final que a IA Recebe

```
System Prompt Completo:
=======================

# `execute_code`: Execute Python or JavaScript code...
# `web_search`: Current Date: 2026-04-02...
# `file_search`: Use the `file_search` tool to find relevant information...

You are a helpful AI assistant specialized in data analysis.
You have access to code execution, web search, and document analysis tools.
When a user asks you to:
- Analyze data or write code: use execute_code
- Find current information: use web_search
- Extract information from uploaded documents: use file_search

Always prefer using tools over just explaining what you would do.
```

## 🔄 Como Adicionar um Novo Prompt?

### Opção 1: Modificar Descrição de Ferramenta Existente

**Para execute_code**:
1. Abrir: `api/server/services/Files/Code/process.js`
2. Encontrar: Função `primeCodeFiles()`
3. Mudar: A string de `toolContext` que retorna

**Para web_search**:
1. Abrir: `api/app/clients/tools/util/handleTools.js`
2. Encontrar: Linhas 335-349
3. Mudar: A string de `toolContextMap[tool]`

**Para file_search**:
1. Abrir: `api/app/clients/tools/util/fileSearch.js`
2. Encontrar: Linhas 45-65
3. Mudar: A string de `toolContext`

### Opção 2: Adicionar Instruções via Agent Config

**Via Database**:
```javascript
db.agents.updateOne(
  { _id: "seu_agente_id" },
  {
    $set: {
      "instructions": "Você é um especialista em análise de dados...",
      "additional_instructions": "Ao executar código:\n1. Sempre use pandas\n2. Crie gráficos...",
    }
  }
)
```

**Via `librechat.yaml`**:
```yaml
agents:
  meu_agente:
    name: "Analista de Dados"
    model: gpt-4
    promptPrefix: "Você é um especialista..."
    additional_instructions: "Ao executar código..."
```

### Opção 3: Adicionar Nova Ferramenta com Prompt

1. **Criar a ferramenta** (Tool constructor com schema + description)
2. **Adicionar em `handleTools.js`**:
   ```javascript
   if (tool === Tools.minha_ferramenta) {
     toolContextMap[tool] = "Meu prompt customizado...";
     requestedTools[tool] = async () => {
       // Lógica de loading
     };
   }
   ```
3. **Adicionar em `agent.tools`**:
   ```javascript
   if (ephemeralAgent?.minha_ferramenta === true) {
     tools.push(Tools.minha_ferramenta);
   }
   ```

## 🎯 Fluxo de Descoberta Resumido

```
┌──────────────────────────────────┐
│ Agent é carregado com:           │
│ tools = ['execute_code', ...]   │
└────────────┬─────────────────────┘
             │
┌────────────▼──────────────────────────────────────┐
│ handleTools.js para cada tool:                    │
│ - Cria objeto com schema + description           │
│ - Salva description em toolContextMap[tool]      │
└────────────┬──────────────────────────────────────┘
             │
┌────────────▼──────────────────────────────────────┐
│ run.ts monta system prompt:                       │
│ systemMessage = toolContextMap.values.join('\n') │
│ systemContent = systemMessage + instructions    │
└────────────┬──────────────────────────────────────┘
             │
┌────────────▼──────────────────────────────────────┐
│ LLM recebe:                                       │
│ - system: systemContent (com descrições)         │
│ - tools: [schema1, schema2, ...]                 │
│ - user: "O que você precisa?"                    │
└────────────┬──────────────────────────────────────┘
             │
┌────────────▼──────────────────────────────────────┐
│ LLM decide: qual ferramenta usar + argumentos    │
│ → tool_call(name, arguments)                    │
└──────────────────────────────────────────────────┘
```

---

## ✅ Conclusão

- **Descoberta**: Agent.tools → loadTools() → toolContextMap
- **Descrição**: Hardcoded em handleTools.js ou gerado dinamicamente
- **Escolha**: LLM vê description + schema → decide usar se relevante
- **Execução**: tool_call → graph.invoke → resultado retorna

**Se você quer mudar um prompt**: edite `handleTools.js` ou a função de "prime" da ferramenta.
