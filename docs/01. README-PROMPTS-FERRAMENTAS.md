## ⚡ TL;DR (Muito Longo; Não Leu)

```
Agent.tools = ['execute_code', ...]
  ↓
loadTools() cria [CodeTool, WebSearchTool, ...]
  ↓
toolContextMap = { 
  'execute_code': 'Execute Python/JS...',
  'web_search': 'Search the web...',
  ...
}
  ↓
systemContent = toolContextMap.values.join('\n') + agent.instructions
  ↓
LLM recebe: tools[] + systemContent (com descrições)
  ↓
LLM decide qual tool usar
  ↓
Tool é executada (POST /exec para sandbox)
  ↓
LLM responde ao usuário
```

## ❓ FAQ

### P: Por que a IA não está usando o sandbox?
**R**: Veja `FLUXO-DESCOBERTA-FERRAMENTAS.md` → Checklist → Execute_code não está em `agent.tools`?

### P: Como mudo o prompt do execute_code?
**R**: 
1. Edite `api/server/services/Files/Code/process.js`
2. Função `primeCodeFiles()`
3. Mude `toolContext`

### P: Onde está o prompt exato do web_search?
**R**: `api/app/clients/tools/util/handleTools.js` linhas 335-349. Ver `PROMPTS-ATTUAIS-FERRAMENTAS.md`.

### P: Como adiciono uma nova ferramenta?
**R**: Leia `FLUXO-DESCOBERTA-FERRAMENTAS.md` seção 7. Depois `SNIPPETS-CODIGO-PROMPTS.md` seção "Como Editar".

### P: Como edito instruções do agente?
**R**: Via DB (`db.agents.updateOne`), ou via `librechat.yaml`.

### P: O que é toolContextMap?
**R**: Dicionário que mapeia `tool_name` → `description_string`. Usado para montar system prompt.
